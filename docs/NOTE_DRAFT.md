## 本文

### はじめに
AIで、案も文章もUIもコードも、速く大量に作れるようになりました。
でも実際にやってみると、しんどいのは「作る」ことではありませんでした。

本当に重かったのは——
**どれを採用するか。なぜそれが使えるか。安全か。そして、次にどこから戻るか。**

その「判断」と「再開」を支えるために、小さな個人用ダッシュボード **Shizuku OS Dashboard v0.1** を作りました。

### 作ったもの
1画面に、4つのカードをまとめています。

- **今日の制作候補**：今日いちばん進める1つを決めて、見失わない
- **夜タスク3行ログ**：やった／学び／次やる。1日を3行で残す
- **制作中リンク**：Figma・GitHub・メモ・公開URLを置いて、再開を速くする
- **Quality Gate**：案や制作物を「採用／保留／捨てる」で判定する

データはこの端末（localStorage）にだけ保存され、外部には送りません。

公開URL：https://shizuku-os-dashboard.vercel.app/

### 核にしたのは「Quality Gate」
このツールでいちばん作りたかったのは、Quality Gate でした。

AIは案をいくらでも出してくれます。だからこそ、出てきた案を**そのまま通さない**仕組みが要る。
Quality Gate では、案を4つの視点で見ます。

- 部下視点（実装できる／運用が重すぎない／壊れにくい）
- 消費者視点（分かりやすい／欲しいと思える／違和感が少ない）
- 勝算（強みがある／自分のブランドに合う／次に使える）
- 安全（個人情報OK／権利OK／戻せる）

そのうえで、採用・保留・捨てる、を選ぶ。**最終的に決めるのは、AIではなく自分**です。

### 作り方で意識したこと
- 思想（何のために作るか）→ ルール → 実装、の順で進めた
- AIには「作業のルール」を先に渡してから実装させた（CLAUDE.md）
- 非プログラマーでも扱えるよう、READMEを起動方法・未実装・安全まで整えた
- 量産っぽい賑やかなUIにせず、冷色と明朝で静かに保った
- ルーティングも保存も、余計な依存を足さず軽く済ませた

### 技術
React / TypeScript / Tailwind CSS / Vite / localStorage / Vercel。
設計から実装・ドキュメント・公開まで、一人で担当しました。

### まだやっていないこと
ログイン・外部API連携・通知・課金・複数ユーザー対応は、v0.1では入れていません。
まず「迷わず戻れる4カードが、ローカルで確実に動く」ことを完成の線にしました。

### おわりに
AIに全部任せるための道具ではなく、**AIを使いながら、自分の判断と美意識を失わないため**の道具。
そういうものを、これからも小さく作っていきます。
